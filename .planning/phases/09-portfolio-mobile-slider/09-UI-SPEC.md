## Breakpoints

This phase introduces a formal three-tier breakpoint contract. All mobile-specific rules are additive
CSS under `@media (max-width: 767px)`. Tablet rules sit under `@media (768px–1023px)`.
Desktop (≥ 1024px) is the existing layout — no changes permitted there.

| Tier | Range | Layout Mode |
|------|-------|-------------|
| Mobile | ≤ 767px | Full-screen stacked card slider |
| Tablet | 768px–1023px | Condensed split-screen (reduced columns, larger touch targets) |
| Desktop | ≥ 1024px | Existing split-screen — **zero changes** |

**Source:** `mobile_context` specification.

## Typography

All fonts, sizes, and weights are carried from the existing `PortfolioPage.css` design system.
Mobile adjustments apply `clamp()` scaling only where existing values overflow at 375px.

**Exactly 4 type sizes are declared for this phase.**

| Role | Size | Weight | Line Height | Font | Notes |
|------|------|--------|-------------|------|-------|
| Kicker / label | 10px | 300 | 1.4 | Geist Mono | Letter-spacing 0.18–0.24em, uppercase — existing |
| Body / prose | 15px | 300 | 1.55 | Inter | Existing `.sections p` rule |
| Tagline / supporting | 18px | 300 italic | 1.3 | Cormorant Garamond | Reduced from 22px desktop for mobile fit. Metric chip `.cn` numbers also render at this size via the heading token at display weight — see note below |
| Heading (h1 + h2 shared) | clamp(36px, 9.5vw, 56px) | 300 | 0.92 | Cormorant Garamond | Single heading token. h1 and h2 are never on screen simultaneously on mobile. Desktop overrides to larger clamp values remain untouched. Metric chip numbers reference this token for the large display treatment — see note below |

**Metric chip number treatment:** `.chip .cn` uses the heading token (`clamp(36px, 9.5vw, 56px)`, weight 300, Cormorant Garamond) at mobile. The chip container constrains the visual size via `min-width: 120px` and the number auto-scales within it. No separate size token is required.

**Weight vocabulary:** 300 (light/regular throughout) + 300 italic (accent/em elements). Two weights only.

**Primary visual anchor:** Project `h2` heading on mobile — largest typographic element in view after the image panel. It is the first major text the eye lands on when the image panel scrolls above the fold.

## Mobile Layout Contracts

### Chrome (Fixed UI Layer)

The chrome layer (`z-index: 80`) contains logo, topmeta, YearNav, ProgressBar, and nav arrows.
On mobile (≤ 767px), chrome is reorganised as follows:

**Logo**
- Keep: `position: fixed; top: 8px; left: 16px` (reduce left from 36px to 16px on mobile)
- Logo image height: 48px on mobile (reduced from 72px)

**Top meta bar (`.topmeta`)**
- Hide: `"Travaillons ensemble →"` CTA pill — too wide for mobile
- Show: availability dot + "Dispo — Q3 2026" text only
- Keep: "Nous contacter" text button (navigates to outro slide)
- Position: `top: 16px; right: 16px`
- Font size: 10px on mobile (kicker token)

**YearNav (`.year-nav`)**
- Hide entirely on mobile: `display: none`
- Replace with: dot indicator row (see Slide Dot Indicator section below)

**ProgressBar (`.bottom`)**
- Mobile: condense to bottom-left, `left: 16px; right: 96px; bottom: 16px`
- Hide: slide title text (`.counter` label) — show counter fraction and progress line only
- Progress line: keep full-width, 1px height

**Nav arrows (`.nav`)**
- Mobile: `bottom: 12px; right: 16px`
- Button size: 44px × 44px (existing — already touch-compliant)
- Gap between buttons: 8px

## Interaction Contracts

### Swipe Gestures (Mobile)

GSAP Observer already handles `wheel`, `touch`, and `pointer` input. No new gesture library.
The existing `go(n)` function drives all navigation.

| Gesture | Action |
|---------|--------|
| Swipe left | `go(currentIndex + 1)` — next slide |
| Swipe right | `go(currentIndex - 1)` — previous slide |
| Tap nav arrow → | `go(currentIndex + 1)` |
| Tap nav arrow ← | `go(currentIndex - 1)` |
| Tap dot indicator | `go(dotIndex)` — direct jump |
| Tap "Nous contacter" in topmeta | `go(slides.length - 1)` — outro slide |

**Observer is already active on the portfolio page.** Phase 9 must NOT re-create or duplicate it. The only change is ensuring the touch input path is not blocked by `overflow: hidden` on a scrollable `.left` panel.

**Scroll-inside-slide guard:** If `.left` text panel on mobile is `overflow-y: auto` (to handle long project content), add `touch-action: pan-y` on the `.left` div. The Observer must detect whether the user is scrolling inside the `.left` panel vs. swiping the whole slide. Pattern: use `onDragEnd` threshold — Observer's `tolerance` (pixels) prevents accidental swipes during text scroll. Existing `GSAP Observer` `tolerance` value should be set to `40px` minimum on mobile (review current config in PortfolioPage.tsx — the `go` function is the callback, so Observer config is where tolerance lives).

### Slide Transition Animation

Existing opacity-crossfade (`DURATION: 0.9s`, `EASE: power2.inOut`) is preserved on mobile.
No new transition type introduced. The transition is already 60fps-tested.

### Touch Target Audit

### Swipe Affordance Indicator

On first load on mobile only (detected via `localStorage.getItem('portfolio-swipe-seen')`):
- Show a one-time fade-in + fade-out hint: a small centered element at the bottom of the first slide
- Copy: `"Glissez →"` (French, mono font, 10px, color: `--ink-dim`)
- Duration: visible for 2s, fades out over 0.5s
- Set `localStorage.setItem('portfolio-swipe-seen', '1')` on first gesture detected
- Implemented as a GSAP `gsap.fromTo()` on a fixed DOM element with `z-index: 95`

### DotNav Component Contract

```
DotNav
  props:
    slides: PortfolioSlideData[]   — the adapted project array (not including cover/outro)
    totalCount: number             — total slide count including cover + outro
    currentIndex: number           — 0-based, including cover at 0
    onGo: (index: number) => void
  render:
    - Wrapper: div.dot-nav (CSS: mobile-only via @media, hidden on desktop)
    - One dot per slide (totalCount dots)
    - Dot 0 = cover, dots 1..N = projects, dot N+1 = outro
    - aria-label on each: "Aller à la diapositive {n}" for cover/outro,
      "Aller au projet {name}" for project slides
    - role="tablist" on wrapper, role="tab" on each dot
```
