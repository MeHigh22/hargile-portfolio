    In the full-motion entry animation's `onComplete` callback (after `ScrollTrigger.refresh()`), add below-fold detection and ScrollTrigger creation:

    ```typescript
    // After ScrollTrigger.refresh():
    const panelEl = panelRef.current!;
    const panelHeight = panelEl.clientHeight;
    const panelRect = panelEl.getBoundingClientRect();

    sections.forEach((section) => {
      const el = section as HTMLElement;
      const rect = el.getBoundingClientRect();
      const relativeTop = rect.top - panelRect.top;

      if (relativeTop >= panelHeight) {
        // Below fold: entry animation set it to opacity:1/y:0 via stagger,
        // but user cannot see it yet. Reset to hidden and attach ScrollTrigger.
        gsap.set(el, { opacity: 0, y: 30 });
        ScrollTrigger.create({
          trigger: el,
          scroller: panelEl,
          start: 'top 85%',
          onEnter: () => {
            gsap.to(el, {
              opacity: 1,
              y: 0,
              duration: 0.5,
              ease: 'power3.out',
            });
          },
        });
      }
    });
    ```

    IMPORTANT: The `sections` variable is already defined earlier in the same gsap.context() scope (line 115: `const sections = panel.querySelectorAll('[data-anim="cs-section"]')`). Reuse it — do NOT re-query.

    IMPORTANT: These ScrollTrigger instances are created INSIDE the gsap.context() callback, so they will be automatically reverted when ctxRef.current.revert() is called in closePanel. No additional cleanup needed.
